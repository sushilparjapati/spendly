❯ what does this project do                                                     

  Read 4 files (ctrl+o to expand)

⏺ Spendly is a personal expense tracker web app built with Flask. Here's what it
   does (and where it stands):

  Purpose: Let users log and track their spending, see category breakdowns
  (Bills, Food, Health, Transport), and understand their spending patterns over
  time.

  Current state — it's a starter/teaching scaffold:
  - Landing page, login, and register pages are built and styled
  - SQLite database layer is stubbed out (students fill it in)
  - Core features are placeholder routes still to be implemented:
    - Add / edit / delete expenses (Steps 7–9)
    - User auth — login, logout, profile (Steps 3–4)
    - Dashboard with spending summaries

  It appears to be a step-by-step student project where you build out the full
  app incrementally.

❯ what tech stack does this project use                                         

  Read 2 files (ctrl+o to expand)

⏺ Backend
  - Python 3.12
  - Flask 3.1 — web framework
  - Werkzeug — utilities (password hashing, routing)
  - SQLite — database (via Python's built-in sqlite3)

  Frontend
  - Jinja2 — HTML templating (comes with Flask)
  - Plain CSS (static/css/style.css)
  - Vanilla JavaScript (static/js/main.js)
  - No frontend framework or build tool

  Testing
  - pytest + pytest-flask

  Tooling
  - uv — package manager
  - Git — version control

  Simple, minimal stack — no ORM, no JS framework, just the essentials.

❯ explain the project structure to me

⏺ expense-tracker/
  │
  ├── app.py                  # Main Flask app — defines all routes
  ├── main.py                 # Entry point (likely just runs app.py)
  │
  ├── database/
  │   ├── __init__.py         # Makes database a Python package
  │   └── db.py               # SQLite helpers: get_db(), init_db(), seed_db()
  │
  ├── templates/              # Jinja2 HTML templates
  │   ├── base.html           # Shared layout (navbar, footer, CSS links)
  │   ├── landing.html        # Public homepage
  │   ├── login.html          # Login form
  │   └── register.html       # Registration form
  │
  ├── static/
  │   ├── css/style.css       # All app styles
  │   └── js/main.js          # Client-side JavaScript
  │
  ├── pyproject.toml          # Project metadata + dependencies (uv)
  ├── requirements.txt        # Same deps in pip format
  ├── uv.lock                 # Locked dependency versions
  └── .python-version         # Pins Python 3.12

  How it fits together:

  - A request hits app.py → the matching route renders a template from
  templates/
  - Templates extend base.html which loads CSS/JS from static/
  - The database/ package handles all SQLite operations — app.py will call these
   when auth and expense features are built out
